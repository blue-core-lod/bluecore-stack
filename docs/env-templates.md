# Repo .env templates

`update-stack --install` uses these as the source for each sibling repo's
local `.env`. For a repo named `<name>` (the checkout directory basename), the
template is `<name>.env`. During `--install`, if the repo is missing a `.env`,
it is generated by copying the matching template; existing `.env` files are
never overwritten.

Repos without a template here (`bluecore-models`, and the `graph-toolbox`
submodule at `nginx/toolbox`) don't use a repo-level `.env` — their config comes
from the compose `environment:` blocks.

Current templates:
- `bluecore-stack.env` — this repo's own root `.env`, seeded during `--install` when it is missing
- `bluecore_api.env`
- `bluecore-workflows.env`
- `marva_editor.env`
- `sinopia_editor.env`
